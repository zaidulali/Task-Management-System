# Product Backlog - TaskSphere

This backlog lists the user stories for the TaskSphere project, categorized by priority.

## High Priority

### Application Features
- **User Registration:** As a new user, I want to register for an account so that I can access the system.
- **User Login:** As a registered user, I want to log in securely so that I can view my private tasks.
- **View Tasks:** As a logged-in user, I want to view my tasks in a list so that I can see what needs to be done.
- **Create Task:** As a logged-in user, I want to create new tasks so that I can keep track of new work.

### DevOps & Infrastructure
- **Environment Variables:** As a developer, I want to use environment variables for configurations so that secrets are not hardcoded.
- **Pull Request Workflow:** As a developer, I want to set up a pull request workflow so that all code modifications are checked before merging.

---

## Medium Priority

### Application Features
- **Edit Task:** As a logged-in user, I want to update my tasks so that I can change their status or details.
- **Delete Task:** As a logged-in user, I want to delete tasks so that I can clean up my list.
- **User Logout:** As a logged-in user, I want to log out so that my account remains secure on shared devices.

### DevOps & Infrastructure
- **Backend Dockerization:** As a DevOps engineer, I want to dockerize the backend so that it runs consistently in all environments.
- **Frontend Dockerization:** As a DevOps engineer, I want to dockerize the frontend so that it is easy to build and serve.
- **CI with GitHub Actions:** As a DevOps engineer, I want to configure GitHub Actions for continuous integration so that linting and tests run on every push.

---

## Low Priority

### DevOps & Infrastructure
- **CircleCI Integration:** As a DevOps engineer, I want to configure CircleCI so that passing builds can be automated for release.
